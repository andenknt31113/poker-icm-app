# ICMの「なんとなく」を、計算で終わらせるアプリを作りました

店舗トーナメントやアマチュアの大会に出ていると、必ずぶつかる壁があります。
バブル間近やファイナルテーブルでのオールイン、コールするべきかフォールド
するべきか——チップEVだけで考えると簡単なのに、ICM（賞金の期待値）が
絡むと途端に判断が難しくなる、あの瞬間です。

「このハンドならコールでいいはず」「でもバブルだから危ないかも」と、
最終的には感覚で決めていた人も多いのではないでしょうか。私自身がそうでした。
ICMIZERのような有名なツールもありますが、英語UIで専門用語も多く、
腰を据えて使うにはハードルを感じていました。

そこで、日本語で気軽に使えるICMの練習アプリ「Poker ICM/BF」を個人で
作りました。この記事では、その紹介と、こだわったポイントをまとめます。

https://poker-icm-app.andenknt31113.workers.dev/

## こんな悩みありませんか

- バブル前後のオールイン判断、いつも自己流
- ICMツールを触ってみたけど英語で用語が難しく続かなかった
- コールかフォールドか、その場の感覚で判断してしまう
- 復習する手段がなく、同じような場面でまた迷う

Poker ICM/BFは、こうした「なんとなく」を計算と反復練習で埋めていく
ことを目指したアプリです。

## アプリの機能

Poker ICM/BFには、大きく5つの機能があります。

### 1. 厳密ICM計算による判定

多くのツールはBubble Factor（BF）による近似で必要勝率を出しますが、
このアプリは「フォールドした場合」「コールして勝った場合」「コールして
負けた場合」の3つの最終スタックをそれぞれ計算し、そのつどICM
（賞金期待値）を計算し直して判定する方式を採用しています。近似ではなく
毎回ICMを引き直すので、ズレが起きにくいのが特徴です。この計算部分は
54件のテストで確認しています。

[スクショ: 判定結果・詳しい計算式]

### 2. 「計算機」ではなく「トレーナー」

数字を入力して結果を眺めるだけのツールにはしたくありませんでした。
このアプリの中心は、ランダムに出題されるシナリオに対してcall/foldを
即座に判断する練習モードです。実戦のプレッシャーに近い形で、繰り返し
判断力そのものを鍛えられます。

[スクショ: 練習画面（call/foldボタン）]

### 3. 導入コース（🎓）

いきなり練習問題を出されても何を学べばいいか分かりにくいので、
バブルの罠やサテライトの掟を数学的に体感できる5問の導入コースを
用意しました。ICMがなぜ重要なのか、体を動かしながら理解できる
構成にしています。

### 4. 成績の推移・復習機能

練習の正解率が推移として見られるので、自分の得意・苦手がなんとなく
分かるようになっています。間違えた問題は復習リストに残るので、
解きっぱなしにせず振り返ることができます。

[スクショ: 成績の推移パネル]

### 5. 全員vs全員 BFヒートマップ

複数人が絡む状況では、誰との組み合わせが特に危険なのか一目では
分かりません。全プレイヤー同士のBubble Factorをヒートマップで
一覧できる機能を用意し、テーブル全体のプレッシャー構造を俯瞰できる
ようにしています。

[スクショ: BFヒートマップ]

## こだわったところ：厳密ICM計算の話

少し技術的な話になりますが、なるべく噛み砕いて説明します。

ICMは「今のチップスタックの状態から、それぞれのプレイヤーが最終的に
獲得する賞金の期待値」を計算する考え方です。オールインの局面では、
本来「フォールドした場合」「コールして勝った場合」「コールして負けた
場合」で全員のスタックが変わるので、それぞれの場合でICMを計算し直す
のが最も正確です。

ただ、これを毎回きちんと計算するのは大変なので、多くのツールは
Bubble Factor（BF）という近似値を使って手早く必要勝率を出します。
近似なので早くて便利なのですが、状況によっては実際の数字と少しズレる
ことがあります。

このアプリでは、練習問題の判定に関しては近似を使わず、3パターンの
最終スタックそれぞれについて律儀にICMを計算し直す方式にしています。
「近似でどれくらいズレるのか」を比較表示する機能もあるので、興味が
あればBF近似との違いも見比べてみてください。計算ロジックについては
54件のテストで細かい数値まで確認しています。
